# How-to Guides

How-to guides are problem-oriented. They assume you know what you want to achieve and provide a direct, verified path to doing it. Each guide targets a specific operational task.

| Guide | Task |
| --- | --- |
| [Manage Repositories](manage-repositories.md) | Create, update, transfer, and delete repositories via the API |
| [Automate Issues](automate-issues.md) | Create issues, assign labels, bulk-update across a repository |
| [Manage Organizations](manage-organizations.md) | List members, manage teams, administer organization settings |
| [Search and Discover](search-and-discover.md) | Query code, repositories, users, and issues with the Search API |
| [Manage GitHub Actions](manage-actions.md) | Repository secrets, workflow dispatches, and self-hosted runners |

Unlike [Tutorials](../tutorials/index.md), how-to guides do not explain the underlying concepts. They are recipes: follow the steps to achieve the stated outcome.
